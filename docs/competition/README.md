# GeoSight Competition Submission Package

This folder is the competition handoff for the current GeoSight build. It assumes a live judged demo first, with a consistent backup package if live providers are slow.

## Core Story

GeoSight should be presented as a **live location-intelligence system** rather than a map plus chatbot. The strongest current story is:

1. **Columbia River infrastructure** for the original high-stakes siting narrative
2. **Tokyo commercial analysis** to prove the platform works outside the Pacific Northwest and to demonstrate honest global coverage behavior
3. **Washington residential due diligence** to show everyday usefulness, school context, and trust-aware analysis

The most current differentiators to emphasize are:

- live multi-source geodata rather than fabricated answers
- deterministic scoring plus evidence-kind labeling
- source awareness with freshness, confidence, and regional fallback context
- new subsurface intelligence: groundwater, soil profile, and seismic design values
- 10-year climate history, not just a weather snapshot
- GeoScribe report generation from the same live context bundle

## Demo Principles

- Lead with the live product, not the deck.
- Let the Columbia River story land before opening secondary supporting cards.
- Use Tokyo to prove GeoSight is global-minded and honest about coverage gaps.
- Use Washington residential to show the product is useful to ordinary people, not only infrastructure judges.
- If a live route is slow, use the fallback banner and backup assets as part of the reliability story rather than pretending nothing happened.
