# Recorded Demo Guide

Use this package only as a fallback if the live demo slows down or loses momentum.

## Recording Plan

- Record the Columbia River infrastructure story first.
- Record the Tokyo commercial story second.
- Record the Washington residential story third.
- Keep the flow aligned with the live demo order so the backup package feels like the same product story, not a different presentation.

## Recording Rules

- Keep each clip short and self-contained.
- Narrate the decision point, not every click.
- Show a recommendation, the supporting evidence, and a trust surface in every clip.
- Include at least one moment where GeoScribe or the report panel is visible, even if only briefly.
- If a provider is slow during recording, let the fallback banner or trust messaging appear instead of cutting around it.

## Export Targets

- `docs/captures/geo-sight-primary-demo.gif`
- `docs/captures/geo-sight-globality.gif`
- `docs/captures/geo-sight-backup-demo.mp4`

## What To Capture

- Columbia River mission objective and final recommendation
- One score or factor-evidence view showing direct live, derived live, and proxy heuristic labels
- One source-awareness or provenance view
- One subsurface or climate-history moment
- One Tokyo globality moment
- One Washington school-context moment
- One GeoScribe report view or report-generation moment

## Delivery Notes

- Use the recorded demo only if the live walkthrough loses momentum.
- Keep the backup assets aligned with the live demo wording so the story feels seamless.
- If only one fallback artifact is needed, prefer the backup video before jumping straight to static screenshots.
