# Demo Scripts

## 90 Seconds

- Open the Columbia River infrastructure story.
- Show the active location, mission score, and one piece of inline provenance.
- Ask for a mission-aware recommendation and point to the direct versus derived evidence labels.
- End by saying GeoSight is a spatial reasoning system, not a generic map or chatbot.

## 3 Minutes

- Start with the Columbia River story and explain why it is the strongest infrastructure comparison.
- Move into the mission run and show the result timeline, evidence mix, and recommendation.
- Open the comparison card and highlight how the score changes with different candidate sites.
- Switch to Tokyo to prove the same workflow works outside the Pacific Northwest.
- Close by showing that every answer is grounded in live source metadata.

## 5 Minutes

- Open with the Columbia River demo and spend time on the mission run, score, and comparison.
- Show source awareness and explain how direct, derived, and proxy signals are labeled.
- Switch to Tokyo and call out the global coverage story.
- Switch to a Washington residential search and show school context plus trust signals.
- Close with the fallback and evidence story: if a provider fails, GeoSight still returns a grounded answer.

## Presenter Notes

- Do not explain every card.
- Stop on the first strong recommendation and then justify it with evidence.
- Keep the demo moving from place to place, not from menu to menu.
- If a card feels too verbose, collapse it and return to the main story.
